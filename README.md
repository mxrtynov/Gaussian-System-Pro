# Распределённая система решения СЛАУ (Gaussian Elimination + TCP Workers)

Приложение демонстрирует полный цикл параллельного решения систем линейных алгебраических уравнений (СЛАУ) методом **исключения Гаусса**. Координатор управляет **TCP-воркерами** (отдельные процессы), каждый из которых выполняет операции исключения строк. Пользовательский интерфейс на **WPF** позволяет генерировать тестовые матрицы, загружать собственные данные, анализировать ускорение и строить графики производительности.

---

## Как всё устроено

```
┌─────────────┐      TCP       ┌─────────────────┐   запуск процессов   ┌────────────────┐
│   WPF UI    │ ─────────────► │  Coordinator    │ ──────────────────►  │ Worker process │
│(Client)     │ ◄───────────── │ (TCP server +   │   (по одному на      │  (port N)      │
└─────────────┘   возвращает   │  управление)    │    порт)             │  ┌────────────┐│
                   решение     └─────────────────┘                      │  │TCP server  ││
                                               │                        │  └────────────┘│
                                               └──────────────────────► └────────────────┘
                                                           ... (2..N workers)
```

1. **WPF Client** формирует задание (путь к матрице, путь к вектору свободных членов) и отправляет координатору.
2. **Координатор**:
   - Загружает  матрицу и вектор из файлов.
   - Запускает `N` процессов `Gaussian.Worker.exe` на разных портах.
   - Распределяет строки матрицы между воркерами (циклически, `rank, rank+total, ...`).
   - Выполняет **прямой ход Гаусса**:
       - Рассылает текущую строку-пивот всем воркерам.
       - Воркеры исключают свои строки и сигнализируют о завершении.
   - Выполняет **обратный ход**: воркеры вычисляют свои компоненты решения и синхронизируют их.
   - Останавливает воркеры, замеряет время, вычисляет ускорение и невязку.
3. **Воркер**:
   - Ожидает соединения от координатора.
   - Получает свою порцию строк матрицы и вектора `b`.
   - Участвует в прямом/обратном ходе, обмениваясь данными через тот же TCP-канал.
4. **Клиент** отображает:
   - Последовательное и параллельное время, ускорение, эффективность.
   - Лог всех операций.
   - Графики сравнения времени и загрузки узлов.
   - Тест производительности при разном количестве воркеров (1,2,4,6,8).

## Состав репозитория

|       Компонент        | Назначение                                                                                                              |
|------------------------| ------------------------------------------------------------------------------------------------------------------------|
| `Gaussian.Shared`      | Общие модели (`SparseMatrix`, `MatrixElement`, `SolveRequest/Response`, `SessionStats`, `WorkerBenchmark`).             |
| `Gaussian.Coordinator` | TCP-координатор (`SolutionCoordinator`), загрузка данных (`DataManager`), генерация случайных систем (`DataGenerator`). |
| `Gaussian.Worker`      | Консольное приложение-воркер, получает строки и участвует в исключении Гаусса по TCP.                                   |
| `Gaussian.Client`      | WPF интерфейс: управление задачами, графики (`ChartRenderer`), сохранение результатов, логирование.                     |
| `Gaussian.Tests`       | Модульные тесты для `DataManager`, `DataGenerator`, `SolutionCoordinator`, `Solver` и всех моделей.                     |
| `Gaussian.Solver`      | `GaussEngine` с последовательной версией Гаусса (используется для эталонного замера).                                   |

# Использование программы

## Главное окно

### 1. Генерация тестовых данных
- **Размер** – введите начальный размер матрицы (например, 100)
- **Кол-во систем** – сколько матриц сгенерировать
- **Плотность** – передвиньте ползунок для выбора процента ненулевых элементов
- Нажмите **«Генерировать»** – файлы сохранятся в `D:\RIS_COURSE_PROJECT\Matrices\`

### 2. Загрузка своих файлов
- **«Выбрать матрицу»** – выберите `.txt` файл с матрицей (числа через пробел или табуляцию)
- **«Выбрать вектор b»** – выберите `.txt` файл со свободными членами (одно число на строку)
- **«Выбрать папку»** – загрузите все пары `matrix_N.txt` / `freeterms_N.txt` из папки

### 3. Запуск решения
- Выберите количество узлов (2, 4, 6, 8) из выпадающего списка
- Нажмите **«ЗАПУСТИТЬ РАСЧЕТ»**
- В логе появятся результаты:
  - Невязка решения
  - Время последовательного и параллельного выполнения
  - Ускорение и эффективность
- После решения программа предложит сохранить результат в файл

### 4. Лог вычислений
- В правой части окна отображается подробный лог
- Показаны все этапы: генерация, загрузка, решение, результаты

## Вкладка «Тест узлов»

Для проверки масштабируемости:
1. Нажмите **«ЗАПУСТИТЬ ТЕСТ ВСЕХ МАТРИЦ»**
2. Программа автоматически прогонит каждую матрицу на 1, 2, 4, 6, 8 узлах
3. Будет построен **график зависимости времени от количества узлов**
4. Каждая матрица отображается отдельной цветной линией
